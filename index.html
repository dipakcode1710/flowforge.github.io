Getting Started

Quick Start

Get a FlowForge application running in three steps.

ℹ️ Requirements

Java 17+, Maven 3.8+. An IDE is optional but recommended.

Step 1 — Build the Framework

Clone the repository and install the framework into your local Maven repository.

$ cd flowforge_framework
$ mvn clean install
…
[INFO] BUILD SUCCESS

Step 2 — Scaffold a New App

$ flowforge create app my-app
$ cd my-app

Step 3 — Run

$ mvn exec:java

📦 Bean Registered: example.UserController
🚀 Controller Ready: example.UserController
 
=================================
🚀 FlowForge Started Successfully

Entry Point

package example;

import flowforge.Flow;

public class AppMain {
    public static void main(String[] args) {
        Flow.run(AppMain.class);
    }
}

Core

Bootstrapping

FlowForge starts via a single call: Flow.run(AppMain.class). This triggers the full startup sequence.

#PhaseDescription
1Classpath ScanningDiscovers all annotated classes in the base package
2Bean RegistrationRegisters controllers, services, and components
3Dependency InjectionResolves and injects constructor dependencies
4Route MappingMaps annotated methods to HTTP paths
5Server StartupStarts the embedded HTTP server

Component Scanning

FlowForge scans the base package of your main class (e.g. example.*) and automatically detects @Controller, @Service, @Component, and @ConfigurationProperties classes.

Core

Routing & Controllers

Controller Example

@Controller
public class UserController {

    @Get("/hello")
    public String hello() {
        return "Hello FlowForge 🚀";
    }

    @Post("/create")
    public String create(@RequestBody String body) {
        return "Created: " + body;
    }
}

Supported Annotations

AnnotationDescription
@ControllerMarks a controller class
@GetHandles GET requests
@PostHandles POST requests
@RequestBodyReads the request body
@ExceptionHandlerHandles exceptions globally

Path Variables

@Get("/user/{id}")
public String getUser(@PathVariable("id") String id) {
    return "User: " + id;
}

Query Parameters

@Get("/search")
public String search(@QueryParam("q") String q) {
    return "Query: " + q;
}

Request Body

@Post("/create")
public String create(@RequestBody String body) {
    return "Created: " + body;
}
Core

Dependency Injection

FlowForge includes basic dependency injection. Constructor injection and field injection are on the roadmap.

Currently Supported

  • Basic bean registration via annotations
  • Type-based bean lookup
  • Manual constructor wiring
⚠️ Not Yet Implemented

@Autowired, constructor injection, and bean scopes are planned for a future release.

Bean Annotations

AnnotationPurpose
@ControllerHTTP handler class
@ServiceBusiness logic class
@ComponentGeneric managed bean
@ConfigurationPropertiesConfig-bound bean

Example

@Service
public class UserService {
    public String getData() {
        return "data";
    }
}

@Controller
public class UserController {

    private UserService userService;

    public UserController(UserService userService) {
        this.userService = userService;
    }
}

Injection Lifecycle

  1. Scan all classes in base package
  2. Instantiate beans in dependency order
  3. Register instances in the context
  4. Inject via constructor parameters

Features

Middleware System

Middleware allows pre/post request processing. Common uses: authentication, logging, and input validation.

Using @Auth Middleware

@Auth
@Get("/secure")
public String secure() {
    return "secured";
}

Available Middleware Hooks

  • @Auth — Validates JWT token before the handler runs
  • Logging — Automatic request/response logging
  • Validation — Request body validation

Features

Exception Handling

Define a global exception handler to return consistent error responses across your entire API.

Global Handler

@Controller
public class GlobalExceptionHandler {

    @ExceptionHandler(RuntimeException.class)
    public ErrorResponse handle(RuntimeException e) {
        return new ErrorResponse("ERROR", e.getMessage());
    }
}

Error Response Shape

{
  "error": "RUNTIME_ERROR",
  "message": "Something went wrong"
}

Global CLI Setup (Windows)

Follow these steps to use flowforge from any terminal on Windows.

Step 1 — Build the JAR

$ mvn clean package

Step 2 — Create install folder

C:\flowforge

Step 3 — Copy the JAR

flowforge-core-1.0.jar  →  C:\flowforge\

Step 4 — Create flowforge.bat

@echo off
java -cp "C:\flowforge\flowforge-core-1.0.jar" flowforge.cli.FlowForgeCLI %*

Step 5 — Add to PATH

Add C:\flowforge to your system PATH environment variable, then restart your terminal.

Step 6 — Verify

$ flowforge

Reference

Best Practices

  • Keep controllers thin — only handle HTTP concerns
  • Put all business logic in @Service classes
  • Use DTOs for request/response models — never expose internal entities directly
  • Enable dev tools only when app.env=dev
  • Use environment-specific config files (app-dev.properties, app-prod.properties)
