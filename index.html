Getting Started

Quick Start

Get a FlowForge application running in three steps.

ℹ️ Requirements

Java 17+, Maven 3.8+. An IDE is optional but recommended.

Step 1 — Build the Framework

$ cd flowforge_framework
$ mvn clean install
[INFO] BUILD SUCCESS

Step 2 — Scaffold a New App

$ flowforge create app my-app
$ cd my-app

Step 3 — Run

$ mvn exec:java
 
🚀 FlowForge Framework :: (v1.0)
 
Scanning package: example
Bean Registered: example.UserController
Controller Ready: example.UserController
=================================
FlowForge Started Successfully
=================================
Server Port : 9091
Docs UI : http://localhost:9091/dev/docs-ui

Entry Point

The main class must be annotated with @App. Flow.run() will throw an error if @App is missing.

package example;

import flowforge.Flow;
import flowforge.core.annotations.App;

@App
public class AppMain {
    public static void main(String[] args) {
        Flow.run(AppMain.class);
    }
}
⚠️ @App is Required

Calling Flow.run() on a class without @App throws IllegalArgumentException immediately.

Core

Bootstrapping

FlowForge starts via Flow.run(AppMain.class). This triggers the full startup sequence in order.

#PhaseDescription
0@App CheckValidates the entry class has @App — fails fast if missing
1BannerPrints the FlowForge ASCII banner
2Classpath ScanRecursively discovers all annotated classes in the base package
3Bean RegistrationInstantiates and registers @Component, @Service, @Controller, @ConfigurationProperties
4Dependency InjectionResolves @Inject, @Value, and config bindings for all beans
5Route MappingRegisters @Get/@Post/@Put/@Delete methods from @Controller beans
6Server StartupStarts the embedded HTTP server on the configured port
Core

Routing & Controllers

Controller Example

@Controller
@Tag("User APIs")
public class UserController {

    @Get("/hello")
    public String hello() {
        return "Hello FlowForge";
    }

    @Post("/create")
    public String create(@RequestBody UserRequest req) {
        return "Created: " + req.name;
    }

    @Put("/user/{id}")
    public String update(@PathVariable("id") int id, @RequestBody UserRequest req) {
        return "Updated: " + id;
    }

    @Delete("/user/{id}")
    public String delete(@PathVariable("id") int id) {
        return "Deleted: " + id;
    }
}

HTTP Method Annotations

AnnotationHTTP MethodBody Support
@Get("/path")GETNo
@Post("/path")POSTRequired
@Put("/path")PUTRequired
@Delete("/path")DELETEOptional

Path Variables

@Get("/user/{id}/orders")
public String orders(
    @PathVariable("id") int id,
    @QueryParam("status") String status
) {
    return "User=" + id + ", Status=" + status;
}

Optional Query Parameters

@Get("/filter")
public String filter(
    @QueryParam("category") String category,
    @QueryParam(value="page", required=false) Integer page
) {
    return "Category=" + category + ", Page=" + page;
}

Grouping with @Tag

@Tag can be placed on a controller class or individual method. Method-level tag overrides the class-level tag. Tags are used to group routes in the Docs UI.

@Tag("User APIs")       // all routes in this controller get this tag
@Controller
public class UserController {

    @Tag("Admin APIs")    // overrides class-level tag for this route only
    @Get("/admin")
    public String admin() { return "admin"; }
}
Core

Dependency Injection

FlowForge uses field-based injection. Annotate a field with @Inject and the framework resolves and injects the bean automatically at startup.

@Inject

@Controller
public class UserController {

    @Inject
    private UserService service;

    @Inject
    private ServerConfig config;

    @Get("/app")
    public String app() {
        return service.getAppName();
    }
}

@Value — Config Injection

Inject individual config properties directly into fields. Supports a defaultValue fallback.

@Service
public class UserService {

    @Value(value = "app.name", defaultValue = "DefaultApp")
    private String appName;

    public String getAppName() { return appName; }
}

@ConfigurationProperties — Typed Config

Bind a whole config prefix to a class. Fields are auto-mapped by name.

// app.properties: server.port=9091  server.name=FlowForgeServer

@ConfigurationProperties("server")
public class ServerConfig {
    public int port;
    public String name;
}

// Inject it anywhere with @Inject
@Inject
private ServerConfig config;

Bean Annotations

AnnotationPurpose
@ControllerHTTP handler class — routes registered automatically
@ServiceBusiness logic class
@ComponentGeneric managed bean
@ConfigurationPropertiesConfig-bound bean with prefix mapping

Injection Lifecycle

  1. Scan all classes in base package recursively
  2. Instantiate beans with no-arg constructor
  3. Register instances in Context
  4. For each bean: resolve @Inject fields, inject @Value fields, run ConfigBinder
  5. If a dependency isn't found in context, it's created on-demand and registered

Features

Middleware & AOP

FlowForge supports per-route middleware via AOP-style annotations. Any class implementing the Middleware interface can be attached to a route.

Middleware Interface

public interface Middleware {
    default void   before(RequestContext ctx) {}
    default void   after(RequestContext ctx, Object result) {}
    default Object around(RequestContext ctx, MiddlewareChain next) throws Exception {
        return next.proceed(ctx);
    }
}

AOP Annotations

AnnotationHookWhen it runs
@Before(MyMiddleware.class)before()Before the controller method
@After(MyMiddleware.class)after()After the controller method
@Around(MyMiddleware.class)around()Wraps the entire execution
ℹ️ Execution Order

Chain order: @Around → @Before → controller → @After. If the same class is used in multiple annotations, it is added to the chain only once.

Custom Middleware Example

public class LogMiddleware implements Middleware {

    @Override
    public void before(RequestContext ctx) {
        System.out.println("BEFORE: " + ctx.exchange.getRequestURI());
    }

    @Override
    public void after(RequestContext ctx, Object result) {
        System.out.println("AFTER: " + result);
    }
}

public class TimeMiddleware implements Middleware {

    @Override
    public Object around(RequestContext ctx, MiddlewareChain next) throws Exception {
        long start = System.currentTimeMillis();
        Object result = next.proceed(ctx);
        System.out.println("Time: " + (System.currentTimeMillis() - start) + "ms");
        return result;
    }
}

Attaching Middleware to a Route

@Before(LogMiddleware.class)
@Around(TimeMiddleware.class)
@Get("/data")
public String data() {
    return "result";
}
Features

Authentication (JWT)

FlowForge has first-class JWT support. Use @Auth on a method or class to require a valid Bearer token.

Protecting Routes

@Auth                   // any valid token
@Get("/secure")
public String secure() { return "protected"; }

@Auth(role = "ADMIN")   // token must carry role=ADMIN
@Get("/admin")
public String admin() { return "admin only"; }

@Auth                   // class-level: all routes require auth
@Controller
public class SecureController { ... }

Generating a Token

String token = JwtUtil.generateToken("dipak", "ADMIN");
// pass as: Authorization: Bearer <token>

Auth Behaviour

ConditionResponse
No Authorization header401 Missing or invalid token
Invalid / expired token401 Invalid or expired token
Wrong role403 Forbidden: Insufficient role
Valid token + correct role200 — proceeds to controller
✅ Disable Auth in Dev

Set auth.enabled=false in app-dev.properties to bypass JWT checks during development.

⚠️ Security Note

Never hardcode JWT secrets in source files. Use environment-specific config files.

Features

Validation

Annotate controller method parameters directly. Validation runs before the method is invoked and throws a RuntimeException on failure.

Available Constraints

AnnotationTargetBehaviour
@NotNullAny parameterFails if value is null
@Min(n)Numeric parameterFails if value < n
@Max(n)Numeric parameterFails if value > n

Example

@Get("/user")
public String getUser(
    @QueryParam("id") @NotNull @Min(1) Integer id,
    @QueryParam("age") @Min(1) @Max(120) Integer age
) {
    return "id=" + id + ", age=" + age;
}

Validation Error Response

Validation failures are caught by the exception handling system. Define a handler for RuntimeException to return a clean JSON error.

{
  "error": "RUNTIME_ERROR",
  "message": "Parameter 'id' must be >= 1"
}
Features

Exception Handling

Define a global exception handler in any @Controller class. Use @ExceptionHandler to map exception types to handler methods, and @ResponseStatus to control the HTTP status code.

Global Handler

@Controller
public class GlobalExceptionHandler {

    @ResponseStatus(500)
    @ExceptionHandler(RuntimeException.class)
    public ErrorResponse handleRuntime(RuntimeException e) {
        return new ErrorResponse("RUNTIME_ERROR", e.getMessage());
    }

    @ResponseStatus(400)
    @ExceptionHandler(IllegalArgumentException.class)
    public ErrorResponse handleBadRequest(Exception e) {
        return new ErrorResponse("BAD_REQUEST", e.getMessage());
    }
}

How It Works

  • Exact exception type is matched first, then parent type
  • @ResponseStatus sets the HTTP status code — defaults to 500 if omitted
  • If no handler is registered, a generic JSON error with status 500 is returned

Error Response Shape

{
  "error": "BAD_REQUEST",
  "message": "Parameter 'id' is invalid"
}

Architecture Overview

Startup Flow

Flow.run(AppMain.class)
   ↓
@App check → fail fast if missing
   ↓
ClassScanner → recursive scan of base package
   ↓
Context → register beans (@Component, @Service, @Controller, @ConfigurationProperties)
   ↓
Injector → resolve @Inject, @Value, ConfigBinder for all beans
   ↓
Dispatcher → map @Get/@Post/@Put/@Delete methods + AOP annotations
   ↓
Server.start(port) → embedded HttpServer

Request Lifecycle

Incoming HTTP request
   ↓
Route matching (path variables extracted)
   ↓
AOP middleware chain (@Around → @Before)
   ↓
AuthMiddleware (if @Auth present)
   ↓
invokeController()
  ├── Resolve @QueryParam / @PathVariable / @RequestBody
  ├── Run Validator (@NotNull, @Min, @Max)
  └── Invoke controller method
   ↓
AOP @After hook
   ↓
JSON / plain text response

Reference

Best Practices

  • Always annotate your entry class with @App
  • Keep controllers thin — only handle HTTP concerns
  • Put all business logic in @Service classes
  • Use DTOs for request/response models — never expose internal entities directly
  • Use @ConfigurationProperties for grouped config instead of many @Value fields
  • Define a GlobalExceptionHandler with @ResponseStatus on every handler
  • Use environment-specific config files — never hardcode ports or secrets
  • Set auth.enabled=false in app-dev.properties to speed up local testing
